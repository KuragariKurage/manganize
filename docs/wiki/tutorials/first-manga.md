# 初めてのマンガを生成する

**対象**: Manganize Web App を使い始めたばかりのユーザー
**目的**: 最初のマンガ画像を生成する手順を学ぶ
**所要時間**: 約 10 分

このチュートリアルでは、Manganize Web App を使って、トピックから「まんがタイムきらら」風 4 コマ漫画を生成する基本的な流れを学びます。

---

## 前提条件

以下がセットアップ済みであることを確認してください：

- Manganize がインストールされている（[Getting Started](getting-started.md) を参照）
- Web サーバーが起動している（`task dev` または `uv run fastapi dev web/main.py`）
- ブラウザで `http://127.0.0.1:8000` にアクセスできる

---

## ステップ 1: メインページにアクセスする

1. ブラウザで `http://127.0.0.1:8000` を開きます
2. Manganize のホームページが表示されます
3. 以下の要素が画面に表示されていることを確認：
   - **トピック入力エリア**: 「マンガにしたい内容を入力してください」
   - **キャラクター選択**: ドロップダウンで「くらげ (kurage)」が選択されている
   - **生成ボタン**: 「マンガを生成」ボタン

---

## ステップ 2: トピックを入力する

トピック入力エリアに、マンガにしたい内容を入力します。

### 例: Transformer アーキテクチャについて

```
Transformerアーキテクチャについて
```

### 入力のヒント

- **短い質問形式**: 「○○について」「○○とは何か」
- **URLも可能**: 記事やブログのURL（エージェントが自動的にコンテンツを取得）
- **長文も可能**: 最大 50,000 文字まで対応

---

## ステップ 3: キャラクターを選択する

デフォルトでは、**くらげちゃん** が選択されています。

- 初回はそのままで OK です
- 後でカスタムキャラクターを作成することもできます（[カスタムキャラクターの作成](../how-to/custom-character.md) を参照）

---

## ステップ 4: マンガを生成する

1. **「マンガを生成」ボタンをクリック**します
2. 進捗表示エリアが表示され、リアルタイムで生成状況が更新されます：

   **進捗状態の流れ**:
   - 🔍 **リサーチ中**: トピックに関する情報を収集しています
   - ✍️ **シナリオ作成中**: 4 コマ漫画の脚本を書いています
   - 🎨 **画像生成中**: マンガ画像を生成しています
   - ✅ **完了**: マンガ画像が表示されます

3. 生成には **3〜5 分** ほどかかります（トピックの複雑さにより変動）

---

## ステップ 5: 生成結果を確認する

生成が完了すると、以下が表示されます：

- **生成されたマンガ画像**（4 コマ形式、9:16 アスペクト比）
- **自動生成されたタイトル**（トピックから 3〜5 単語で要約）
- **ダウンロードボタン**: PNG 形式でダウンロード可能
- **フルサイズ表示**: 画像をクリックするとモーダルで拡大表示

### 画像をダウンロードする

1. **「ダウンロード」ボタンをクリック**
2. ファイル名は自動的に `manganize_20251228_123456_transformer_architecture.png` 形式で保存されます

### 画像を拡大表示する

1. **生成された画像をクリック**
2. フルサイズモーダルが開きます
3. モーダルの外側をクリック、または ESC キーで閉じます

---

## ステップ 6: 別のトピックで試してみる

もう一度トピックを入力して、別のマンガを生成してみましょう。

### おすすめのトピック例

- 「量子コンピュータについて」
- 「コーヒーの淹れ方」
- 「マイクロサービスアーキテクチャとモノリシックアーキテクチャの違い」
- 「宇宙の起源について」

---

## トラブルシューティング

### 生成ボタンがクリックできない

- トピック入力エリアが**空白**になっていませんか？
- 入力エリアに何か入力すると、ボタンが有効になります

### 進捗が途中で止まった

- ブラウザの**コンソール**を開いて、エラーメッセージを確認してください（F12 キー）
- SSE 接続が切断された場合、自動的に再接続を試みます
- エラーが表示された場合は、「再試行」ボタンをクリックしてください

### 画像が生成されない

- **GOOGLE_API_KEY** が正しく設定されていますか？
  ```bash
  echo $GOOGLE_API_KEY
  ```
- API キーが有効でない場合は、[Google AI Studio](https://aistudio.google.com/app/apikey) で新しいキーを取得してください

### 進捗状態が「エラー」になった

- エラーメッセージを確認してください
- 以下の原因が考えられます：
  - トピックがマンガ化に適していない（関連度が低い）
  - API レート制限に達した
  - ネットワークエラー
- 「再試行」ボタンで再度生成を試みてください
