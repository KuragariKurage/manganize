# Explanation（解説）

**理解指向** - 設計思想と背景を理解するためのドキュメント。

## 利用可能な解説

### [アーキテクチャ解説](architecture.md)

- 3 段階パイプライン構成
- 各コンポーネントの役割
- 状態管理とデータフロー

### [設計の意思決定](design-decisions.md)

- LangGraph 採用の理由
- Gemini 3 Pro Image Preview の選択
- uv によるパッケージ管理
- 型ヒントの徹底

### [プロンプトエンジニアリング解説](prompt-engineering.md)

- 3 つのプロンプトの構成
- 設計原則（具体性、構造化、制約の明示）
- デバッグ手法

## 読む順序

```
1. アーキテクチャ解説 - システム全体の構造
     ↓
2. 設計の意思決定 - 技術選定の背景
     ↓
3. プロンプトエンジニアリング解説 - プロンプト設計
```
